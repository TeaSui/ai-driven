# impl-13: Audit Trail (4.2)

**Status:** 🔲 To Do  
**Priority:** 4.2  
**Impact:** Full compliance and traceability of AI-generated code

---

## Goal

- Log all AI-generated code diffs to a dedicated S3 bucket
- Record prompt + response pairs for compliance
- Add a "generated by AI" footer to all PRs

## Proposed Changes

### Audit S3 Bucket
- [ ] Create `ai-driven-audit-trail-{accountId}` S3 bucket in CDK
- [ ] Enable versioning and server-side encryption
- [ ] Set lifecycle: transition to Glacier after 90 days, expire after 1 year
- [ ] Structure: `audit/{year}/{month}/{ticketKey}/{timestamp}/`

### Prompt-Response Recording
- [ ] Create `AuditService` in `core` module
- [ ] After Claude invocation, store:
  | File | Content |
  |------|---------|
  | `prompt.txt` | System prompt + user message |
  | `response.txt` | Raw Claude response |
  | `metadata.json` | Model, tokens, duration, ticket info |
- [ ] Add `AUDIT_BUCKET` env var to CDK stack

### PR Footer
- [ ] Append AI disclosure footer to all PR descriptions:
  ```
  ---
  🤖 This pull request was generated by AI (model: claude-opus-4-6)
  Ticket: PROJ-123 | Generated: 2024-01-15T10:30:00Z
  ```
- [ ] Update `PrCreatorHandler` to include footer in PR body

### Code Diff Storage
- [ ] Store generated file diffs in audit bucket
- [ ] Include before/after comparison where applicable

## Testing Strategy
